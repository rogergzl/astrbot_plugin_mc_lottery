# astrbot_plugin_mc_lottery 使用指南

## 插件简介

`astrbot_plugin_mc_lottery` 是一款为 Minecraft 服务器设计的抽奖插件。支持**个人抽奖**和**全局开奖**两种模式，每轮每人最多中奖一次。奖励通过 `mrcon` 插件在服务器上自动执行，实现 QQ 群与 Minecraft 双向联动。

## 核心机制：抽奖轮次

每轮抽奖中，**每个玩家最多只能中奖一次**。重复抽奖会提示本轮已中的奖品。

- **个人抽奖**：用户发送 `/抽奖`，仅对该用户执行一次抽奖判定，结果仅自己可见
- **全局开奖**：管理员 `/开奖` 对所有已开启代理抽奖的用户进行一轮抽奖，广播中奖名单
- **自动开奖**：按设定间隔自动执行全局开奖
- **自动下一轮**：开奖后 X 小时自动清除本轮记录并触发新一轮开奖
- **重置**：管理员 `/抽奖重置` 清空本轮中奖记录，相当于立即开启新轮

## 命令列表

### 用户命令

| 命令 | 说明 |
|------|------|
| `/抽奖绑定 <MC_ID>` | 绑定 QQ 与 Minecraft ID |
| `/自动抽奖` | 切换代理抽奖（参与全局开奖） |
| `/抽奖自动兑换` | 切换中奖后自动兑换 |
| `/抽奖` | 个人抽奖（每轮每人最多中一次，重复抽提示已中的奖品） |
| `/抽奖兑换` | 手动领取一条待发奖品 |
| `/抽奖帮助` | 显示全部命令帮助 |

### 管理命令（需管理员权限）

| 命令 | 说明 |
|------|------|
| `/抽奖开启` | 开启抽奖并启动自动开奖 |
| `/抽奖关闭` | 关闭所有抽奖 |
| `/开奖` | 立即全局开奖，显示当前轮次所有中奖者（含个人抽奖） |
| `/抽奖重置` | 归档本轮记录并清除，开启新一轮 |
| `/抽奖间隔 <分钟>` | 设置自动开奖间隔（1-1440） |
| `/抽奖间隔小时 <小时>` | 设置开奖后几小时自动下一轮（0=关闭） |
| `/抽奖在线 开\|关` | 仅在线玩家参与开关 |
| `/抽奖列表` | 查看绑定详情与中奖状态 |
| `/抽奖历史 [轮次数]` | 查看历史开奖记录（默认仅管理员，可配置为全员） |

## 配置说明

插件配置位于 `_conf_schema.json`，支持在 AstrBot 管理面板可视化编辑。

### 基础配置

| 配置项 | 类型 | 说明 |
|--------|------|------|
| `bind_file` | string | 绑定数据存储路径 |
| `super_admin` | string | 超级管理员 QQ 号（所有群有效） |
| `default_auto_interval` | int | 默认自动开奖间隔（分钟） |
| `default_round_interval_hours` | int | 全局开奖后几小时自动下一轮（0=关闭） |
| `max_history_rounds` | int | 保留最近多少轮历史记录（0=不保留） |
| `history_admin_only` | bool | 是否仅管理员可查看历史记录（默认开启） |
| `default_only_online` | bool | 初始是否仅在线玩家可参与 |

### 奖品配置 `default_prizes`

按奖项等级分类，可视化表单编辑：

| 奖项 | prizes | probability | count | command |
|------|--------|-------------|-------|---------|
| 特等奖 | 钻石x64 | 0.05 | 1 | `give {player} minecraft:diamond 64` |
| 一等奖 | 附魔金苹果x1 | 0.15 | 2 | `give {player} minecraft:enchanted_golden_apple 1` |
| 二等奖 | 铁锭x16 | 0.3 | 3 | `give {player} minecraft:iron_ingot 16` |
| 参与奖 | 经验瓶x16 | 0.5 | 5 | `give {player} minecraft:experience_bottle 16` |

### 分群配置

- `group_prizes`：JSON 字符串，按群号配置独立奖品池，未配置使用 `default_prizes`
- `group_admins`：JSON 字符串，按群号配置管理员白名单

## 管理员权限

以下任一满足即为管理员：
1. `super_admin` 配置的 QQ 号
2. `group_admins` 中对应群号的 QQ 列表

## 注意事项

1. 依赖 `mrcon` 插件执行 MC 命令
2. 未绑定 QQ 的用户无法参与抽奖
3. 绑定数据保存在 `bind_file` 路径，请勿手动删除
